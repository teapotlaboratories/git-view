# Workspace: rename the phone menu "View" + move the tree toggle into it

Asked for: rename the "Git" dropdown to "View", and move the file-tree toggle into that dropdown.

## Change (`ui/Screens.kt`)
- The right-side dropdown chip is now **"View"** on phone/e-ink (where it carries the Files/Chat switch)
  and stays **"Git"** on tablet (its menu is only git actions — no View section). Content-desc follows:
  `view menu` vs `git actions`.
- The explorer-tree ⇄ editor toggle (was a separate `☰` `IconButton` in the bar) is now a **"File tree"**
  item inside the View section, checked when the tree is showing. It appears only in the phone Files pane
  with a file open (`activePane == FILES && ui.activeFile != null`). The bar drops the icon.
- Menu order (phone): **View** → Files / Chat / File tree · **Git** → Working-tree diff / Staged diff /
  History… / Commit… / Push.

## Verified on-device (all three)
- **Phone** (`kancil_test`): chip reads **View**; no tree icon in the bar; menu shows View (Files ✓ /
  Chat / File tree) + Git actions. `2026-07-24-view-menu-phone.png`.
- **Tablet** (`tabS8`): chip stays **Git** (permanent tree + both panes → no View section). Single bar,
  three-pane split unchanged. `2026-07-24-view-menu-tablet.png`.
- **Bigme e-ink** (`bigmeB7`, E-Ink profile ON): chip **View**, same menu on the paper palette.
  `2026-07-24-view-menu-eink.png`.

## Note
On-device verification cost a full re-pair of the phone (uninstalling the debug build to install the
release APK wiped the paired connection; the Base-URL field is pre-filled with a placeholder-looking
default that must be cleared before typing, which tripped up the first attempts). Tablet + e-ink kept
their pairing via `install -r` of the same-signature debug build.
